# lvMosaic 仕様書

## 0. ドキュメント概要

このドキュメントは、Claude Code / Codex などのAIコーディングエージェントにそのまま渡して実装を進めるための仕様書である。

本アプリは、動画に対して手動でモザイク範囲を指定し、フレームごとまたはキーフレームごとに位置・サイズ・回転を調整し、モザイク加工済みのmp4動画を出力するための、超軽量・超高速なクロスプラットフォーム対応ネイティブアプリである。

---

# 1. アプリ概要

## 1.1 アプリ名

正式名称: `lvMosaic`

読み方: `れべるもざいく`

## 1.2 コンセプト

`lvMosaic` は、動画モザイク編集だけに特化した超軽量・超高速ネイティブアプリである。

一般的な動画編集ソフトのような多機能編集は行わず、以下の作業だけを快適に行えるようにする。

1. mp4動画をドラッグ＆ドロップで読み込む
2. モザイク範囲を手動で指定する
3. フレーム移動に合わせてモザイク範囲を調整する
4. 必要に応じてモザイク範囲を回転する
5. 動画の最初/最後の指定フレーム数を除去する
6. 元ファイル名に `_mosaic` を付けたmp4を出力する

## 1.3 最重要方針

このアプリでは、以下を最優先する。

1. 起動が速いこと
2. UI操作が軽いこと
3. フレーム送りが軽いこと
4. モザイク範囲の手動指定が快適であること
5. 出力処理が安定していること
6. 機能を増やしすぎないこと

## 1.4 実装しない機能

以下は初期版では実装しない。

- 自動モザイク追尾
- AIによる対象検出
- カット編集
- テロップ編集
- BGM追加
- 音声編集
- 複数動画の結合
- 複数トラック編集
- 高度なエフェクト
- クラウド連携
- アカウント機能

本アプリはあくまで「モザイク専用ツール」である。

# 2. 推奨技術構成

## 2.1 基本構成

推奨技術スタックは以下とする。

| 項目 | 技術 |
|---|---|
| 言語 | Rust |
| GUI | egui |
| ウィンドウ管理 | winit または eframe |
| 描画/GPU処理 | wgpu |
| 動画デコード | FFmpeg系ライブラリ、またはFFmpegプロセス連携 |
| 動画エンコード | FFmpeg |
| プロジェクト保存 | JSON |
| 対応OS | Windows / macOS / Linux |

## 2.2 技術選定理由

`lvMosaic` は超軽量・超高速なネイティブアプリを目指すため、WebViewベースのUIではなく、RustネイティブのGUI/描画構成を優先する。

eguiはRust製の即時モードGUIライブラリで、ネイティブ環境でも動作する。  
wgpuはRust向けのクロスプラットフォームなグラフィックスAPIで、Vulkan / Metal / D3D12 / OpenGLなどのバックエンドで動作する。

FFmpegは動画のデコード・エンコード・フィルタ処理に利用できる。FFmpegのフィルタ機能はlibavfilterによって提供され、複数入力・複数出力を持つフィルタグラフを構成できる。

## 2.3 回転対応の方針

回転対応は **案A** を採用する。

つまり、モザイク処理はFFmpegのfiltergraphだけに任せず、Rustアプリ側でフレーム画像に対して処理する。

採用方針:

- デコード: FFmpeg
- モザイク処理: Rust側
- 回転矩形マスク処理: Rust側
- プレビュー描画: Rust + wgpu
- エンコード: FFmpeg

理由:

- 回転矩形に対応しやすい
- プレビューと出力結果を一致させやすい
- フレームごとの位置・サイズ・回転指定に対応しやすい
- 将来的なGPU最適化がしやすい

# 5. 先頭/末尾フレーム除去機能

## 5.1 概要

動画出力時に、元動画の最初または最後の指定フレーム数を除去して出力できる機能を実装する。

用途:

- 動画の冒頭に不要な数フレームがある場合
- 動画の末尾に不要な数フレームがある場合
- 生成動画や録画動画の前後の乱れを簡単に除去したい場合

## 5.2 UI項目

右側パネルまたは出力設定パネルに以下を表示する。

```text
出力範囲
開始側除去フレーム数: [ 0 ]
終了側除去フレーム数: [ 0 ]
```

## 5.3 パラメータ

| 項目 | 内容 |
|---|---|
| trim_start_frames | 先頭から除去するフレーム数 |
| trim_end_frames | 末尾から除去するフレーム数 |

## 5.4 初期値

```json
{
  "trim_start_frames": 0,
  "trim_end_frames": 0
}
```

## 5.5 バリデーション

以下の条件を満たす必要がある。

```text
trim_start_frames >= 0
trim_end_frames >= 0
trim_start_frames + trim_end_frames < total_frames
```

不正な場合は出力開始できない。

エラーメッセージ例:

```text
除去フレーム数が動画の総フレーム数以上になっています。
値を小さくしてください。
```

## 5.6 出力対象フレーム

以下の範囲を出力対象とする。

```text
output_start_frame = trim_start_frames
output_end_frame = total_frames - trim_end_frames - 1
```

例:

```text
total_frames = 1000
trim_start_frames = 10
trim_end_frames = 20
```

出力対象:

```text
10 ～ 979
```

出力フレーム数:

```text
970
```

## 5.7 モザイクキーフレームとの関係

プロジェクト内部のキーフレーム番号は、元動画基準のフレーム番号で保持する。

出力時のみ、除去後のフレーム範囲に合わせて処理する。

つまり、以下のようにする。

- 編集画面: 元動画のフレーム番号で管理
- プロジェクト保存: 元動画のフレーム番号で保存
- 出力処理: trim_start_frames ～ total_frames - trim_end_frames - 1 のみ処理
- 出力動画の0フレーム目: 元動画の trim_start_frames フレームに対応

これにより、編集情報が除去設定に依存しない。

# 7. モザイク範囲仕様

## 7.1 モザイク範囲の基本

モザイク範囲は回転可能な矩形オブジェクトとして扱う。

各モザイク範囲は、開始フレーム・終了フレーム・キーフレーム情報を持つ。

## 7.2 モザイク範囲のプロパティ

| 項目 | 内容 |
|---|---|
| id | モザイク範囲ID |
| name | 表示名 |
| start_frame | 有効開始フレーム |
| end_frame | 有効終了フレーム |
| mosaic_size | モザイク粒度 |
| keyframes | キーフレーム配列 |

## 7.3 キーフレームのプロパティ

| 項目 | 内容 |
|---|---|
| frame | フレーム番号 |
| center_x | 中心X座標 |
| center_y | 中心Y座標 |
| width | 幅 |
| height | 高さ |
| rotation_deg | 回転角度 |

座標は動画の元解像度基準で保持する。

## 7.4 座標方式

モザイク範囲は中心座標方式で管理する。

```json
{
  "center_x": 400.0,
  "center_y": 240.0,
  "width": 160.0,
  "height": 90.0,
  "rotation_deg": 0.0
}
```

理由:

- 回転処理がしやすい
- 拡大縮小の基準が明確
- プレビューと出力処理で座標計算を共有しやすい

# 9. キーフレーム仕様

## 9.1 基本仕様

モザイク範囲は複数のキーフレームを持つ。

キーフレーム間は線形補間する。

補間対象:

- center_x
- center_y
- width
- height
- rotation_deg
- mosaic_size

## 9.2 自動キーフレーム作成

以下の操作を現在フレームで行った場合、現在フレームにキーフレームがなければ自動作成する。

- 移動
- リサイズ
- 回転
- mosaic_size変更

## 9.3 線形補間

現在フレームが2つのキーフレーム間にある場合、各値を線形補間する。

```text
t = (current_frame - prev.frame) / (next.frame - prev.frame)
value = prev.value + (next.value - prev.value) * t
```

## 9.4 範囲外の扱い

- 最初のキーフレームより前: 最初のキーフレーム値を使用
- 最後のキーフレームより後: 最後のキーフレーム値を使用
- start_frameより前: モザイク非表示
- end_frameより後: モザイク非表示

# 12. モザイク処理仕様

## 12.1 処理方式

モザイク処理はRust側で行う。

1. FFmpegで対象フレームをデコードする
2. Rust側でフレーム画像を受け取る
3. 現在フレームで有効なモザイク範囲を取得する
4. 各モザイク範囲の補間済み位置・サイズ・回転角度を算出する
5. 回転矩形マスクを作成する
6. マスク範囲内にピクセル化モザイクを適用する
7. 処理済みフレームをFFmpegへ渡す
8. FFmpegでmp4へエンコードする

## 12.2 ピクセル化モザイク

モザイク処理は以下の方式とする。

1. 対象範囲をブロック単位に分割する
2. 各ブロックの代表色を計算する
3. ブロック全体を代表色で塗る

代表色は初期版ではブロック左上のピクセル、または平均色とする。

高速化優先なら左上ピクセルでよい。

## 12.3 回転矩形マスク

回転矩形の判定は、対象ピクセルを矩形のローカル座標へ逆回転して判定する。

擬似コード:

```text
dx = pixel_x - center_x
dy = pixel_y - center_y

local_x =  dx * cos(-theta) - dy * sin(-theta)
local_y =  dx * sin(-theta) + dy * cos(-theta)

inside =
  abs(local_x) <= width / 2 &&
  abs(local_y) <= height / 2
```

insideがtrueのピクセルにのみモザイクを適用する。

## 12.4 処理対象範囲の最適化

全画面を毎回走査しない。

回転矩形の外接矩形を計算し、その範囲内のみ処理する。

```text
bbox_min_x
bbox_min_y
bbox_max_x
bbox_max_y
```

これにより高速化する。

## 12.5 複数モザイク範囲

複数のモザイク範囲が同じフレームに存在する場合は、登録順に処理する。

重なり部分は後から処理したモザイクが上書きしてよい。

```text
動画ファイル名.lvMosaic.autosave.json
```

保存タイミング:

- 30秒ごと
- モザイク範囲追加時
- キーフレーム追加時
- 出力開始時
- アプリ終了時

# 16. パフォーマンス方針

## 16.1 基本方針

- 起動時に重い処理をしない
- 動画読み込み時に全フレームを展開しない
- 必要なフレームのみデコードする
- プレビューは低解像度キャッシュを使う
- 出力時のみ元解像度で処理する
- AI処理や自動追尾を入れない
- UIスレッドを重い処理で止めない

## 16.2 フレームキャッシュ

プレビュー用にフレームキャッシュを持つ。

キャッシュ対象:

- 現在フレーム
- 前後数フレーム
- キーフレーム周辺
- シーク後の周辺

キャッシュ枚数設定:

| モード | キャッシュ枚数 |
|---|---:|
| 低メモリ | 30 |
| 標準 | 120 |
| 高速 | 300 |

初期値は `標準`。

## 16.3 UIスレッドと処理スレッド

以下を分離する。

- UIスレッド
- デコードスレッド
- エンコードスレッド
- フレームキャッシュ管理スレッド

出力処理中もUIが固まらないようにする。

# 20. 実装時の注意

## 20.1 まず操作感を優先する

このアプリの価値は、エンコード機能よりも「手動モザイク指定の快適さ」にある。

そのため、実装順序としては以下を優先する。

1. フレーム送りの軽さ
2. モザイク矩形操作の軽さ
3. キーフレーム補間の分かりやすさ
4. 出力の正確さ

## 20.2 UIを増やしすぎない

設定項目を増やしすぎると、軽量ツールとしての良さが消える。

初期版では以下だけでよい。

- 動画読み込み
- 再生/停止
- フレーム送り
- モザイク作成/編集
- 先頭/末尾フレーム除去
- 出力

## 20.3 自動追尾は絶対に入れない

この仕様では自動追尾は実装しない。

理由:

- 重くなる
- 誤検出対応が必要になる
- GPU/AIモデル依存が増える
- アプリの思想から外れる
- 開発コストが跳ね上がる

`lvMosaic` は「自動で何でもやるツール」ではなく、「手動指定が爆速なツール」である。

# 22. 完成イメージ

`lvMosaic` は以下のようなアプリを目指す。

```text
mp4をD&D
↓
動画が即表示される
↓
対象範囲をドラッグしてモザイク作成
↓
フレームを進めながら位置・サイズ・回転を調整
↓
必要なら先頭/末尾の不要フレーム数を指定
↓
Export
↓
sample_mosaic.mp4 が出力される
```

このアプリは、総合動画編集ソフトではない。

`lvMosaic` は、モザイク編集だけを最速で終わらせるための専用工具である。
